What it does

Super Globals Plus is a developers' tool that makes evaluating variables easy. It produces a clear overview of the contents of strings, arrays, constants, objects, arrays within object, objects within arrays or whatever. Because the amount of information given by this tool was overwhelming to most new users of the Super Globals Plus script, version 1.24 and later only shows $_POST and $_SESSION by default.

Features:

  • Show all GLOBALS, or just the Superglobals you need to evaluate.
  • Show all defined constants (v1.2).
  • Show a list of included files (v1.22).
  • Show Superglobals either in a javascript popup window or at the end of each rendered page. (v1.4)
  • Optionally hide HTTP_ variables when showing all globals.
  • Show the whole content of each variable, including all objects and arrays etc!
  • Recursion protection (optional: recursion level/off)
  • Show the variable type (array, string, object, integer, NULL, boolean, resource).
  • Converts strings into their htmlentities before output (v1.0)
  • Clear output (see screenshot to the right).
  • Easy install on Zen Cart 1.3.9, 1.5.0 and 1.5.1, instructions included.
  • Optional IP address protection.
  • Admin switches for all options.

How to use it

After you have you have copied the required files and installed the SQL patch in your store’s admin, you will see the Configuration->Superglobals selection. Within that selection are the values that you can use to control the add-on’s display.

Installation

There are no core-file overwrites for the catalog area in this mod, but there are optional admin changes that are overwrites. You should always backup your cart’s database and files prior to making any changes.

  1. Run either the superglobals.sql file if you are running Zen Cart version 1.5.0 (or later) or the superglobals_139.sql file if you are running Zen Cart version 1.3.9x. Go to your admin’s Tools->Install SQL Patches; from there you can either upload the .sql file or copy and paste the contents of the file.
  2. Copy these files to your cart, after renaming the "YOUR_TEMPLATE" directories to match your custom template’s name and the "YOUR_ADMIN" directory to match the name of your admin directory:
    1. /includes/stylesheet_superglobals.css
    2. /includes/functions/extra_functions/superglobals.php
    3. /includes/templates/YOUR_TEMPLATE/common/tpl_main_page.php
    4. /includes/templates/YOUR_TEMPLATE/css/stylesheet_superglobals.css
    5. /YOUR_ADMIN/includes/languages/english/extra_definitions/superglobals.php

Optional admin Installation

If you want to use the Superglobals display to debug admin pages, you will need to perform a few more file copies (after renaming the "YOUR_ADMIN_OPTIONAL" directory to match the name of your admin directory); note that there are core file changes:

  1. /YOUR_ADMIN_OPTIONAL/index.php
  2. /YOUR_ADMIN_OPTIONAL/includes/footer.php
  3. /YOUR_ADMIN_OPTIONAL/includes/stylesheet_superglobals.css
  4. /YOUR_ADMIN_OPTIONAL/includes/stylesheet.css
  5. /YOUR_ADMIN_OPTIONAL/includes/functions/extra_functions/superglobals.php

Un-install

Run either superglobals_remove.sql (for Zen Cart v1.5.0 or later) or superglobals_remove_139.sql (for Zen Cart v 1.3.9x) to remove the database entries, then delete the files you previously copied (replacing the admin core files that were overwritten).
